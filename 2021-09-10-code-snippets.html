Displaying Code Snippets:

For our website, we want to be able to display code snippets in our blog posts. To do this you cannot simply just type the code and have it formatted nicely within the post automatically. Instead, you have to use <code> tags around the section of code you want to display. The tag will display the code within the tags as text, but ignore the formatting. This is because browsers ignore line breaks and multiple white spaces. To solve this issue, we can use <pre> tags around the <code> tags that are already around the code snippet. These tags will keep the formatting of the code snippet correct and looking nice.


How to use the Tags:

How your code looks:

<pre>

<code> 

    
      body {
      background-color: red;
      border-radius: 4px;
      }

</code>

</pre>


How it is displayed: 

    
      body {
      background-color: red;
      border-radius: 4px;
      }

Syntax Highlighting with Prism.js:

Sometimes when displaying long code snippets, they can be difficult to understand and read. By using syntax highlighting, it is possible to make the code snippets more readable for the viewer. We can do this easily by using a Javascript library to automatically highlight the code. Two commonly used syntax highlighters are Prism.js and Highlight.js. For the purpose of simplicity, we will focus on Prism.js but the process of using Highlight.js is very similar.

On the Prism website (prismjs.com), you can download the CSS and JS for Prism. They give you options on what languages and themes you plan on using. Once you have selected your options and configured Prism to your needs, download and add the files into your project folder. These files will be used to highlight the syntax of your snippet.

To use these files, you will need to include them in your site. Include the files as I have done below:


<!DOCTYPE html>

<html>

<head>

...

<link href="themes/prism.css" rel="stylesheet" />

</head>

<body>

...

<script src="prism.js"> </script>

</body>

</html>


Prism only works and highlights syntax of items surrounded by <code> tags to encourage good authoring practices. In addition to the <code> tag, Prism recommends you define your code language within the code tag (see the example below). You also can add the <pre> tag from earlier to keep your formatting.

Example of defining class in the code tag:

<code class=“language-css”>

</code”>


Now you know how to download and use a syntax highlighter for code snippet implementation for the website. For more help installing or using the prism.js library, check out their website (prismjs.com).


Manually Highlighting with CSS

If you want to change the colors of your code you can do that with CSS. In between the <code> tags you can add a <div> tag with a class attribute that with CSS and the div tag will disappear in your code segments.

For example:


      <style>

      .changeColor { 

         color: red; 

      } 

      </style>

      <div class="changeColor">body</div> { 

         background-color: blue; 

     }

Will look like:

 
body
 { 

     background-color: blue; 

  }
